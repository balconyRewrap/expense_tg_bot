CRUD Users Config

CRUD operations for user configurations in the database.

async database.crud.user_configs.add_user_config(tg_id: int, language: str, currency: str) → None

Asynchronously adds a user configuration to the database.

This function adds a new user configuration for a given Telegram user ID, language, and currency. If the user does not exist, it raises a UserNotFoundError. If there is an integrity error (such as a unique constraint violation), it raises a UniqueDublicateError.

Parameters:

  • tg_id (int) – The Telegram user ID.

  • language (str) – The preferred language of the user.

  • currency (str) – The preferred currency of the user.

Raises:

  • UserNotFoundError – If the user with the given Telegram ID does not exist.

  • UniqueDublicateError – If there is an integrity error while adding the user configuration.

async database.crud.user_configs.user_config_exist_by_id(session: AsyncSession, tg_id: int) → bool

Check if a user configuration exists in the database.

This asynchronous function queries the database to determine if a user configuration exists for a given Telegram user ID.

Parameters:

  • session (AsyncSession) – The SQLAlchemy asynchronous session to use for the query.

  • tg_id (int) – The Telegram user ID to check for an existing configuration.

Returns:

True if a user configuration exists for the given Telegram user ID, False otherwise.

Return type:

bool
